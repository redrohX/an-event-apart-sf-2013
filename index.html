Day one

Talk by Jeffrey Zeldman, author, Designing With Web Standards, 3rd Ed.

Ten Commandments of Modern Web Design

A nice starter talk where Zeldman gives us ten commandments of modern web design and development. All commandments are obviously based on facts and data that he gathered from his own projects and experiences. Some of the commandments sound quite obvious, but if you think about it, are not always executed in our own projects at Spil.

  • Thou shalt entertain
  • Test everything (including assumptions)
  • Thou shalt iterate
  • Thou shalt ship
  • Engage the community
  • Love thy user as thyself
  • Remember the content and keep it holy
  • Thou shalt make magic
  • Thou shalt prioritize
  • To thine own self be true
Talk by Samantha Warren, Designer, Twitter

Faster Design Decisions with Style Tiles

A talk about a way to deliver designs to clients by using style tiles. Designers have been doing this a certain way for many years (by delivering images or a multitude of images to the client) but this can be done in a much more future proof way by creating these so called style tiles. For Spil Games this could be a useful technique to explore during the project of the new responsive site.

  • Style Tiles are a design deliverable consiting of fonts, colors and interface elements that communicate the essence of a visual brand for the web.
  • Moodboards are too vague
  • Mockups work for really low budgets or sites that are being redefined, rather than redesigned. Works if expectations are set.
  • Style Tiles seperate style and structure
  • Style Tiles are device width agnostic
  • Design is about problem solving
  • Style tile process: Listen, Interpret, Define visual language, iterate
  • www.StyleTil.es

  • Examples on how to create complex interaction designs for touch screens
  • Write/read multi-device web. How do we design for multi-device creation?
  • One handed use, focused flows, just in time actions, cross device usage
  • Cross device usage: access, flow, control, push
Talk by Ethan Marcotte, Author, Responsive Web Design

The Map & The Territory

In this session Ethan revisited what we’ve learned about responsive design, and ensures our content is readily accessible wherever and whenever our users are.

A talk interesting for Spil because we can create nice responsive sites, the content we host on our sites is incompatible with different devices. Ethan doesn't get into this specific kind of problem, but he does go through a couple of different techniques on how to set up content on responsive sites in terms of performance.

  • By keeping your client side code small and lightweight, you can open your product up to new markets.
  • Trouble in responsive paradise?
  • A big issue we face with responsive sites is their performance
  • Sites can be anything between 50kb and 30mb nowadays.
  • Website weight average: 320kb in 2009, 1.6mb in 2013.
  • More people in Africa have access to a mobile phone than electricity.
  • Performance technique: Use SVG's as images
  • Performance technique: cutting the mustard (BBC technique, check if site is modern, if not show really basic version)
  • Selectively displaying data (hide data -- not remove -- if there is no room or bandwidth for it).
  • There are responsive image techniques we can use today already.

Talk by Chris Coyier, Author, CSS-Tricks.com

Preprocessing is for Everybody

In this presentation Chris looked into how preprocessors fit into modern web development. He gave us tips and tricks to convince the boss and colleagues, best practices for using preprocessors, and how to squeeze out all the benefits we can get from them. Still interesting enough for Spil, even if we're already using Sass in the new architecture.

  • Stepping up our abstraction allows us to manage that complexity.
  • Preprocessing is an abstraction. It makes working on websites easier.
  • Besides Sass there is Stylus and Less. Sass is most robust language. Biggest active community.
  • Preprocessing HTML and JS is cool, but not as powerful as CSS preprocessing.
  • Sass simplifies development and lowers the complexity of our site.
  • Every Sass tool is built for more consistency.
  • Make the sites faster with concatenated and compressed CSS.
  • Sass learning period is short. You don't need to rewrite everything right away.
  • Transitioning to Sass is easy. All .css files are valid .scss files.
  • With a little DevOps, writing in Sass can be easier than writing CSS.

Learn practical, effective, efficient techniques that work within any design and development process to give you invaluable insight into your users and what they need and value.

  • Why research? You assumptions are not the same as the real world.

  • Wrong: We don't have the expertise.
  • Wrong: We're already A/B testing.
  • Wrong: We can just build a prototype.
  • Wrong: Steve Jobs didn't need to do research.

Day three

Workshop by Luke Wroblewski, CEO & Co-Founder of Input Factory Inc.

The Web Everywhere – Multi-Device Web Design 1/3

Luke's workshop wasn't really a hands-ons session because of the huge number of attendees. It was more like a college session where Luke explains that the web no longer starts and ends on our desktop. Designing for a cross/multi-device world requires new ways of thinking and doing business.

This was a really useful presentation. Luke managed to give a very interesting six hour presentation on building sites for multi-device web. We shouldn't focus on specific devices, but on our users who are using multiple devices. This is already happening and focusing on one or a couple of specific experiences is not the way to go forward.

  • Samsung alone already has about 30 different screen sizes available in their touch products.
  • The way we interact with our devices is changing (touch, mouse, voice).
  • Resolutions are different across all devices (so don't target on resolution).
  • Support a continuum of screens
  • Account for high resolution
  • Optimize for touch
  • Support cursors & keyboard
  • Smartphones & tablets use overtook PC use around 2011
  • Design and build mobile first
Workshop by Luke Wroblewski, CEO & Co-Founder of Input Factory Inc.

The Web Everywhere – Multi-Device Web Design 2/3

  • Good news: You only need one web design: a multi-device design.
  • Bad news: it's a new & different way of doing web design.
  • Responsive web design: fluid grids, flexible images, media queries.
  • Navigation elements: start with mobile considerations.
  • Navigation elements: adapt as more screen space is available.
  • Navigation elements: support touch ergonomics.
  • Start with mobile.
  • Enhance upward.
  • Build fluidly.
  • Screen continuum: Responsive web design, breakpoints, multi-device design patterns, avoid mobile stacks.
  • Create content-based break points. Not based on resolution.
  • "The more you tweak based on a squishy browser window the less testing for specific devices you'll need."
  • Use multi-device layout patterns.
Workshop by Luke Wroblewski, CEO & Co-Founder of Input Factory Inc.

The Web Everywhere – Multi-Device Web Design 3/3

  • Use high resolution images where possible (there are solutions you can already use).
  • Use scalable vector graphics (SVG) for icons, logo's and every images that's not a photo.
  • SVG's can have media queries inside.
  • Performance is important. 86% of responsive sites send same content to mobile.
  • RWD forces you to think about what's really needed on your page.
  • 40% of users will abandon your site after 3 seconds of waiting it to load.
  • Manage performance: perceived performance, client-side enhancements, server-side components.
  • Reduce requests & file size.
  • Take advantage of HTML5 etc.
  • Perception of fast performance is important.
  • Initial state: Ajax-Include content. Start lightweight and accesible without JavaScript.
  • Expanded state: Ajax-request referenced content and inject it into the page.
  • Set a max performance "budget".
  • Responsive design + server side optimization = RESS
